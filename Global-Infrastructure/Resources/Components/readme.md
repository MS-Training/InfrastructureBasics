# Components Folder

The `Components` folder contains modules that represent specific Azure resources, each defined with the appropriate API and version. These modules are designed to encapsulate the configuration and deployment logic for individual Azure services, ensuring consistency and reusability across environments.

Each component is tailored to the requirements of the environment being deployed using the Global-Infrastructure Framework. This approach allows for modular, maintainable, and scalable infrastructure-as-code, making it easier to manage and update Azure resources as needed.
